WvWAnalyst

A local-first Guild Wars 2 WvW analysis app for batch-processing ArcDPS logs through a WvWAnalyst-enabled Elite Insights build.

The app does not require D:\GW2\EICode. Any root works when the repos keep the expected sibling layout.

Root Layout

<root>\
  GW2-Elite-Insights-Parser\
  WvWAnalyst\

Set your root once in PowerShell:

$Root = 'D:\GW2\EICode'   # Change this to your root if different.
$WvWAnalystRoot = Join-Path $Root 'WvWAnalyst'
$EiRoot = Join-Path $Root 'GW2-Elite-Insights-Parser'

Files You May Need To Edit

Main config

<root>\WvWAnalyst\apps\api\WvWAnalyst.Api\appsettings.json

Edit for storage paths, EI paths, log directories, and authentication.

NuGet config

<root>\WvWAnalyst\NuGet.Config

Usually leave alone. Edit only for different package sources.

Patch metadata

<root>\WvWAnalyst\storage\patch-metadata.json

Usually edit through the Manage tab.

Comp Helper config

<root>\WvWAnalyst\storage\comp-helper-config.json

Usually edit through the Analysis Comp Helper UI.

Auth users

<root>\WvWAnalyst\storage\auth-users.json

Do not hand-edit normally. Use the user CLI.

Audit logs

<root>\WvWAnalyst\storage\audit\audit-yyyy-MM.jsonl

View from the Manage tab Activity Log.

Required Dependencies

  • Windows with PowerShell 5.1 or newer.
  • Git.
  • A modern browser.
  • .NET SDK 10.0 for WvWAnalyst.
  • .NET SDK 8.0, or compatible .NET 8 targeting support, for Elite Insights.
  • The WvWAnalyst-enabled EI fork with WvWAnalystBuilder.cs and SaveOutAnalystJSON.
cd $Root
git clone https://github.com/Opep64/GW2-Elite-Insights-Parser.git
git clone https://github.com/Opep64/WvWAnalyst.git

Build Elite Insights

Default EI CLI path:

<root>\GW2-Elite-Insights-Parser\GW2EI.bin\Debug\CLI\GuildWars2EliteInsights-CLI.exe
dotnet build (Join-Path $EiRoot 'GW2EIParserCLI\GW2EIParserCLI.csproj') -c Debug

If EI is elsewhere, update Workspace:ParserCliPath in <root>\WvWAnalyst\apps\api\WvWAnalyst.Api\appsettings.json.

Build WvWAnalyst

$env:DOTNET_CLI_HOME = Join-Path $Root '.dotnet'
$env:DOTNET_SKIP_FIRST_TIME_EXPERIENCE = '1'
$env:DOTNET_NOLOGO = '1'
$env:APPDATA = Join-Path $Root '.dotnet\appdata'
$env:NUGET_PACKAGES = Join-Path $Root '.dotnet\packages'
$env:MSBuildEnableWorkloadResolver = 'false'

dotnet build (Join-Path $WvWAnalystRoot 'apps\api\WvWAnalyst.Api\WvWAnalyst.Api.csproj') `
  --configfile (Join-Path $WvWAnalystRoot 'NuGet.Config') `
  -p:RestoreIgnoreFailedSources=true

Start, Check, And Stop

Start

& powershell -ExecutionPolicy Bypass -File (Join-Path $WvWAnalystRoot 'tools\scripts\start-local.ps1')

Status

& powershell -ExecutionPolicy Bypass -File (Join-Path $WvWAnalystRoot 'tools\scripts\status-local.ps1')

Stop

& powershell -ExecutionPolicy Bypass -File (Join-Path $WvWAnalystRoot 'tools\scripts\stop-local.ps1')

Useful start options:

& powershell -ExecutionPolicy Bypass -File (Join-Path $WvWAnalystRoot 'tools\scripts\start-local.ps1') -Url http://127.0.0.1:5078
& powershell -ExecutionPolicy Bypass -File (Join-Path $WvWAnalystRoot 'tools\scripts\start-local.ps1') -NoBuild

Foreground run:

& powershell -ExecutionPolicy Bypass -File (Join-Path $WvWAnalystRoot 'tools\scripts\run-local.ps1')

Configuration

Main config file:

<root>\WvWAnalyst\apps\api\WvWAnalyst.Api\appsettings.json

Relative paths in appsettings.json are resolved from <root>\WvWAnalyst\apps\api\WvWAnalyst.Api. Therefore ..\..\..\storage resolves to <root>\WvWAnalyst\storage.

"Workspace": {
  "ParserPath": "..\\..\\..\\..\\GW2-Elite-Insights-Parser",
  "ParserCliPath": "..\\..\\..\\..\\GW2-Elite-Insights-Parser\\GW2EI.bin\\Debug\\CLI\\GuildWars2EliteInsights-CLI.exe",
  "PendingDirectoryPath": "..\\..\\..\\storage\\incoming",
  "ArchiveLogDirectoryPath": "..\\..\\..\\storage\\logs"
}

Import Workflow

  1. Start WvWAnalyst.
  2. Open the URL printed by the start script, usually http://127.0.0.1:5078.
  3. Use the Manage tab.
  4. Drop or select .evtc, .zevtc, or .zip files.
  5. Run Process only new files or Process all and rebuild catalog.

WvWAnalyst creates a temporary EI config using DetailledWvW=true, ParseCombatReplay=true, SaveOutHTML=true, SaveOutJSON=false, and SaveOutAnalystJSON=true.

Authentication

Authentication is controlled in <root>\WvWAnalyst\apps\api\WvWAnalyst.Api\appsettings.json:

"Authentication": {
  "Enabled": true,
  "UsersPath": "..\\..\\..\\storage\\auth-users.json",
  "CookieName": "WvWAnalyst.Auth"
}

Set Enabled to true to require login, or false to disable login. Restart WvWAnalyst after editing the file.

$env:Authentication__Enabled = 'true'
& powershell -ExecutionPolicy Bypass -File (Join-Path $WvWAnalystRoot 'tools\scripts\start-local.ps1')

Manage Users

$ApiDll = Join-Path $WvWAnalystRoot 'apps\api\WvWAnalyst.Api\bin\Debug\net10.0\WvWAnalyst.Api.dll'

dotnet $ApiDll users list
dotnet $ApiDll users add creig
dotnet $ApiDll users reset-password creig
dotnet $ApiDll users remove creig

Password hashes are stored in <root>\WvWAnalyst\storage\auth-users.json.

Activity Log

Audit files are stored at:

<root>\WvWAnalyst\storage\audit\audit-yyyy-MM.jsonl

The Manage tab includes an Activity Log section with a refresh button.

Storage Notes

Durable storage includes storage\fights, storage\logs, storage\patch-metadata.json, storage\comp-helper-config.json, storage\auth-users.json, and storage\audit.

Transient storage includes storage\cache and storage\incoming.

Stop WvWAnalyst before copying storage. storage\auth-users.json and storage\audit can contain sensitive local information. Do not publish them.

Troubleshooting

& powershell -ExecutionPolicy Bypass -File (Join-Path $WvWAnalystRoot 'tools\scripts\status-local.ps1')

Get-Content (Join-Path $WvWAnalystRoot 'storage\cache\wvw-analyst.log') -Tail 80
Get-Content (Join-Path $WvWAnalystRoot 'storage\cache\wvw-analyst.error.log') -Tail 80

If WvWAnalyst cannot find EI, rebuild EI and verify <root>\GW2-Elite-Insights-Parser\GW2EI.bin\Debug\CLI\GuildWars2EliteInsights-CLI.exe.

If auth is enabled and you cannot log in, run dotnet $ApiDll users reset-password creig.
